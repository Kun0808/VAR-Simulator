# VAR Simulator：裁判压力室

> 一个球迷为爱发电的小游戏。不盈利，不商用，纯粹因为太多次对着屏幕喊"这球都看不出来？"决定自己做一个。

每个看球的周末，我们都经历过那种时刻：VAR 房间里的裁判盯着回放看了三分钟，最后做出了一个让你想把遥控器砸向电视的判罚。这个游戏让你坐进那把椅子——你不再是沙发上的吐槽担当，你是那个要在 8 秒内拍板的人。

10 个回合，每个都是似曾相识的争议场景。越位线画在哪儿、滑铲先碰到球还是先碰到人、那个球到底是打在肩膀还是手臂上——你以为是常识的东西，在倒计时压力下会变得没那么确定。判完之后，球迷弹幕会飘过屏幕，教练会拍桌子，媒体会起标题，和你每个周末看到的一模一样。

## 玩法

游戏共 10 个回合，每回合展示一个用 Canvas 实时绘制的 2D 足球争议场景——越位线、滑铲碰撞、手球高亮、传球路线都会在画面上呈现。你需要在 8 秒倒计时结束前，从四个选项中做出判罚：

- **进球有效**
- **越位**
- **犯规**
- **手球**

判罚后系统立即反馈对错、规则解释、球迷吐槽、教练反应和媒体标题。10 个回合结束后，根据正确数给出最终评级，从"世界级 VAR 裁判"到"建议远离 VAR 房间"共五档。

### 数值系统

每次判罚会影响三项数值，实时显示在页面底部。压力值和争议值会随着错误判罚累积，模拟真实 VAR 裁判在连续争议中越来越坐不住的感觉：

| 数值 | 变化规则 |
|------|----------|
| 正确判罚数 | 判罚正确 +1 |
| 压力值 | 正确 -8，错误 +15，超时 +20（0–100） |
| 争议值 | 正确时按事件少量增加（球迷仍会吵），错误 +15，超时 +20（0–100） |

### 评级标准

| 正确数 | 评级 |
|--------|------|
| 9–10 | 世界级 VAR 裁判 |
| 7–8 | 可靠的裁判 |
| 5–6 | 争议制造机 |
| 3–4 | 赛后道歉声明预备役 |
| 0–2 | 建议远离 VAR 房间 |

## 10 个回合事件

这些场景都改编自真实比赛中反复出现的争议类型，没有使用任何真实球队、球员或赛事名称：

| 回合 | 事件 | 正确判罚 |
|------|------|----------|
| 1 | 前锋直塞破门，启动时越位 | 越位 |
| 2 | 滑铲先踢到人后碰到球 | 犯规 |
| 3 | 禁区内手臂张开挡球 | 手球 |
| 4 | 接球时与最后一名防守球员平行 | 进球有效 |
| 5 | 冲撞已抱球的守门员 | 犯规 |
| 6 | 防守球员回传失误送到越位位置前锋脚下 | 进球有效 |
| 7 | 球击中肩膀（非手臂）后进球 | 进球有效 |
| 8 | 干净滑铲先碰球，进攻球员随后倒地 | 进球有效 |
| 9 | 头球前伸手推开防守球员 | 犯规 |
| 10 | 防线身后接长传单刀破门 | 越位 |

## 关于这个项目

这是一个纯兴趣项目，没有商业目的。游戏里所有的球队、球员、赛事都是虚构的，不使用任何真实俱乐部名称、球员姓名、球衣颜色或赛事 logo。如果你在某个回合里看到了自己主队的影子——那纯粹是因为这类争议确实每个周末都在发生。

技术上也保持纯粹：纯原生 HTML + CSS + JavaScript，零外部依赖，零框架，零图片素材。所有足球场景通过 Canvas 2D API 实时绘制，断网也能运行。选择这个技术栈不是偷懒，是希望这个游戏能双击就玩，不需要装任何东西，也不依赖任何 CDN。

## 文件结构

本项目提供两个版本，功能完全一致：

### 单文件版

```
var-simulator.html    # 所有 HTML/CSS/JS 打包在一个文件，双击即玩
```

### 拆分版

```
index.html            # 页面结构入口
css/
  style.css           # 全部样式
js/
  data.js             # 10 个回合数据 + 弹幕池，修改关卡只动这个文件
  game.js             # Canvas 绘制、动画、倒计时、判罚逻辑、数值系统
```

## 运行方式

**单文件版**：直接双击 `var-simulator.html` 在浏览器中打开即可。

**拆分版**：推荐通过本地服务器打开，避免某些浏览器对 `file://` 协议的限制：

```bash
cd VAR-Simulator
python3 -m http.server 8000
```

然后访问 `http://localhost:8000/index.html`。直接双击 `index.html` 通常也能正常工作。

## 自定义关卡

所有回合数据集中在 `js/data.js`（拆分版）或 `var-simulator.html` 的 `events` 数组（单文件版）中。想加入你自己最意难平的那个争议场景？直接改数据就行：

```javascript
{
  id: 1,                    // 回合编号
  title: "事件标题",
  description: "事件描述文本",
  correctAnswer: "越位",     // 进球有效 / 越位 / 犯规 / 手球
  explanation: "规则解释",
  crowdReaction: "球迷反应",
  coachReaction: "教练反应",
  mediaHeadline: "媒体标题",
  controversyImpact: 2,      // 判罚正确时争议值增加量
  scene: {                   // Canvas 绘制配置
    offsideLine: 340,        // 越位线 x 坐标（可选）
    ball: { x1, y1, x2, y2 },        // 足球起止坐标
    attacker: { x1, y1, x2, y2 },    // 进攻球员跑动起止坐标
    players: [...],          // 静态球员位置
    collision: {...},        // 碰撞效果（可选）
    highlight: {...}         // 高亮区域（可选）
  }
}
```

修改 `events` 数组即可增删或调整回合，无需改动其他代码。

## 兼容性

适配桌面和手机浏览器，响应式布局在小屏幕上自动调整为两列按钮和两列数值面板。Canvas 画面按宽度等比缩放。
